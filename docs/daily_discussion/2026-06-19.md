# Daily Discussion Summary - 2026-06-19

## Purpose
Capture the human-AI collaboration decisions and implementation evidence for the SkillsFuture career pathway MVP.

## Main Decisions
- The MVP is questionnaire-first, not resume/JD-first.
- Telegram plus a generated explainability/action-plan report is the MVP demo surface.
- Resume/JD parsing remains stretch until M0-M7 are complete.
- The questionnaire flow is hybrid: assess current-role baseline first, recommend 3 pathways, then ask 3 to 5 target-gap questions after the user chooses a pathway.
- All skill weights are `1.0` for MVP; future core/important/supporting weighting can be explored later.
- Sector and track are metadata/pathway constraints, not skill-vector dimensions.
- Pathway constraints should be configurable so product owners can decide whether sector/track friction is strict or soft.

## Dataset / Technical Findings
- The repo-local dataset path is `dataset/`, not Downloads.
- M1 normalized the SkillsFuture data into roles, unique skills, role-skill requirements, and skill K&A items.
- Generated M1 counts: 2,030 roles, 2,316 skills, 43,799 normalized role-skill requirement rows, and 150,264 K&A rows.
- The data quality report shows role matching, unique-skill matching, K&A coverage, text proficiency normalization, source traceability, and Data Analyst/Data Scientist sample profiles.
- Raw role-skill audit coverage to unique skills is 99.93%, while the final normalized role-skill feature table has no missing `skill_id` or `unique_skill_title`.

## Product Direction
- The product should show why recommendations are made using dataset logic.
- It should offer actionable pathways instead of overwhelming users with every possible skill gap.
- User-facing outputs should emphasize suitability percentage, top gaps, recommended pathways, and concrete learning actions.

## Architecture Decisions
- Use Python as the core programming language.
- Use a sparse skill vector: dimension = unique skill, value = proficiency level.
- Keep skill suitability separate from pathway fit.
- Use deterministic scoring before any parser/agent layer.
- Use K&A items to turn skill gaps into level-specific action planning.
- Use project-level Codex skills to keep future milestone execution scoped.

## Explainability Decisions
- Every score, pathway, and action plan must trace back to user answers and SkillsFuture dataset rows.
- Reports should show matched skills, gaps, current level, target level, and dataset source fields.
- Parser agents, when added as stretch, should extract evidence only; deterministic scoring remains the final scoring authority.

## Judging / Process Notes
- Judging criteria split roughly 50% process and 50% product, so daily discussion archives are part of the submission evidence.
- Process evidence should show human-AI decision making, tradeoffs, and validation, especially because this is likely a solo project.
- The roadmap is milestone-based and goal-prompt styled to prevent future agents from straying into unrelated implementation.

## Open Questions
- Whether to track large generated CSV outputs in git long term or regenerate them during setup.
- How strict future sector/track pathway constraints should be for the demo.
- Whether to introduce manually tagged skill importance after the deterministic MVP is working.
